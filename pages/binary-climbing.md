## 3つの壁を回避して「10,000要素のタプル」を作れるか?

境界線が見えたので、今度は **3つの壁を全部踏まずに** 巨大な型を組み立てたい。

| 壁 | 1ステップで消費 | 余裕 |
|---|---|---|
| 第1 Depth(100) | 深さ +2 | 48回で頭打ち / **余裕なし** |
| 第2 Count(500万) | 作業量 ×4 | 100万枝で爆発 / **余裕なし** |
| **第3 TailCount(1000)** | 反復 +1(深さも幅もゼロ) | **1000回の予算がまるごと残る** |

→ 第3の壁(ループ)の **1000回の予算** で 10,000要素 を組み立てる

- 普通に書くと 1ループ = +1 → 1000要素で予算切れ
- 1ループあたりの **歩幅を倍々に広げる** = $\log_2 10{,}000 \approx 14$
- 1000の予算のうち **14だけ** 使えば届く(Binary Climbing)

## 動きを追う: L = 10 (= 8 + 2) の場合

二進数で **`10 = 1010(2)`**。1が立っている桁の歩幅だけ `Acc` に**足す**:

| ステップ | 歩幅 `T` (`[...T,...T]`で倍々) | 2進ビット | 足す? | 結果 `Acc` |
|---|---|---|---|---|
| 0 | 1 | 0 | スキップ | 0 |
| 1 | 2 | **1** | 足す (`[...Acc,...T]`) | 2 |
| 2 | 4 | 0 | スキップ | 2 |
| 3 | 8 | **1** | 足す | **10** ✓ |

- 歩幅 `T` は **足す/スキップに関係なく** 毎回 `[...T,...T]` で倍々
- 結果 `Acc` は **足したステップだけ** `[...Acc,...T]` で反映

## 完成版 → 召喚

```ts {all|3-4|10-15}
type BuildTuple<
  L extends number,
  T extends any[] = [any],     // 歩幅(倍々で広がる)
  Acc extends any[] = [],      // 結果(足したぶんのみ)
> =
  Acc["length"] extends L
    ? Acc
    : BuildTuple<
        L,
        [...T, ...T],          // 歩幅は毎回 ×2
        [...Acc, ...T]["length"] extends infer NextL
          ? NextL extends number
            ? NextL extends L | (number & { __brand: "overshoot" })
              ? Acc            // 超える → スキップ
              : [...Acc, ...T] // 安全 → 足す
            : Acc : Acc>;

type UltraSuperVeryLongLongTuple = BuildTuple<10000>; // 14ステップで召喚成功 🎉
```
